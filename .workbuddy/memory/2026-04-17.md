## 云托管踩坑记录（血泪史）
- `container.config.json` 里的 `"dataBaseName": "schedule"` 会导致云托管尝试连接云数据库，数据库不存在时容器启动就崩溃 → 移除 `dataBaseName` 和 `executeSQLs`
- Node.js `app.listen(PORT)` 默认绑定 IPv6 (`::`)，微信云托管健康检查走 IPv4，连不上 → 实例数始终为 0 → 反复 `102002`/`timeout` → 修复：`app.listen(PORT, '0.0.0.0')`
- Alpine 镜像下 sqlite3 原生模块编译失败，需用 `node:20` 完整镜像 + `python3 make g++`
- Kimi API `temperature` 只接受 `1`，其他值返回 400
- 云托管容器内存过小（0.5GB）图片识别时 OOM 崩溃 → 建议至少 1C2G
- 腾讯云托管 → 腾讯 TokenHub (`tokenhub.tencentmaas.com`) 响应约 20-30 秒
